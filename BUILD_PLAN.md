# Build plan: מדריך חופש הגדול 2026 — לוח פעילויות אינטראקטיבי

## מה בונים
עמוד HTML **בודד וסטטי** (קובץ `index.html` אחד, ללא build step, ללא תלויות חיצוניות, ללא רשת) המציג ~108 פעילויות קיץ למשפחות בישראל, עם חיפוש חופשי וסינון. **כל הממשק בעברית, RTL native.** אפשר לפתוח את הקובץ בדאבל-קליק או להעלות לכל static host.

מקור הנתונים: `activities.json` (מצורף). אל תמציא פעילויות — השתמש בקובץ כמו שהוא.

## דרישות טכניות (קשיחות)
- קובץ אחד `index.html`. CSS ו-JS inline. הנתונים מוטמעים כ-JS object בתוך הקובץ (`const DATA = {...}`) — לא `fetch`, כדי שיעבוד גם מ-`file://`.
- `<html lang="he" dir="rtl">`. גופן: system font stack עברי (`"Assistant","Heebo","Segoe UI",Arial,sans-serif`) — בלי Google Fonts/CDN.
- אין localStorage/sessionStorage. state ב-JS בזיכרון בלבד.
- Responsive: mobile-first (המשתמשות מגיעות מאינסטגרם בטלפון). breakpoint אחד ל-desktop (grid רב-טורי).
- Vanilla JS. בלי React/framework.

## מודל הנתונים
כל פעילות: `name, region, category, ages, cost, duration, why, tip` (+ אופציונלי `cost_note`).
- `region` (אזור) — 8 ערכים: צפון, חיפה והכרמל, השרון, מרכז, ירושלים, דרום, אילת, פעילויות בבית.
- `cost` (עלות) — 3 ערכים: חינם, בתשלום, בתשלום סמלי.
- `category` (סוג) — טקסט חופשי (מגוון, לא enum).

## פילטרים (לפי מה שהמשתמשת ביקשה: אזור/קטגוריה)
הסינון המרכזי הוא **אזור** ו**עלות**. בנוסף חיפוש טקסט חופשי.

1. **צ׳יפים לפי אזור** — שורת צ׳יפים (pills) בראש: "הכל" + 8 האזורים. בחירה יחידה. הצ׳יפ הפעיל מודגש. הצג ליד כל אזור ספירה (למשל "מרכז 16").
2. **סינון עלות** — 3 טוגלים/צ׳יפים: חינם / בתשלום סמלי / בתשלום (multi-select — אפשר לבחור כמה). ברירת מחדל: הכל דלוק.
3. **חיפוש חופשי** — תיבת input אחת, מסננת על `name + category + why + tip` (case-insensitive, substring). placeholder: "חיפוש פעילות, מקום או מילת מפתח...".
4. **איפוס** — כפתור "נקה סינון" שמחזיר להכל.

הסינונים מצטברים (AND בין סוגי הסינון, OR בתוך רב-בחירה). עדכון התוצאות **מיידי** בכל שינוי (בלי כפתור "חפש").

## קרדיט ליוצרת (חובה — לא להסיר)
המדריך המקורי נוצר על ידי **מורן בן דוד**. חובה לתת לה קרדיט **גלוי בכל חלק של האתר** — גם ב-header וגם ב-footer — כך שהקרדיט תמיד נגיש בלי קשר לאן גוללים. הקישורים:
- אינסטגרם: `https://www.instagram.com/momibd` (טקסט מוצג: `@Momibd`)
- טיקטוק: `https://www.tiktok.com/@momibd1` (טקסט מוצג: `@Momibd1`)

דרישות:
- הלינקים **קליקבילים**: `<a href="..." target="_blank" rel="noopener noreferrer">`, עם אייקון/טקסט אינסטגרם וטיקטוק.
- **Header** מציג "המדריך מאת מורן בן דוד" + שני הלינקים.
- **Footer** חוזר על הקרדיט + הלינקים.
- להוסיף גם קרדיט קצר ב-**toolbar הדביק** (או mini-bar דביק בתחתית) כדי שהקרדיט נראה גם באמצע גלילה, ולא רק בקצוות העמוד.
- אין להסתיר, להקטין עד בלתי-קריא, או להסיר את הקרדיט בשום מצב — כולל ב-print stylesheet (הקרדיט נשאר גם בהדפסה).

## הבהרת תפקידים (חובה להציג)
יש להבהיר באתר, בטקסט גלוי (למשל ב-footer ליד קרדיט היוצרת), ש:
- **כל התוכן, ההמלצות והטיפים הם של מורן בן דוד** (היוצרת המקורית של המדריך).
- **נתי הררי בנה/ה את האתר בלבד** — במטרה להנגיש את המידע ולהפוך אותו לנוח לחיפוש וסינון. נתי אינו/ה מחבר/ת התוכן.

נוסח מוצע ל-footer (לשנות לפי הצורך):
> כל התוכן וההמלצות מאת **מורן בן דוד** · [@Momibd](instagram) · [@Momibd1](tiktok).
> האתר נבנה על ידי **נתי הררי** כדי להנגיש את המדריך ולאפשר חיפוש וסינון נוחים.

## פריסה
- **Header**: כותרת `המדריך הגדול לחופש הגדול 2026`, תת-כותרת `100 רעיונות למשפחות עם ילדים — מצפון ועד דרום`, וקרדיט **קליקבילי** `מורן בן דוד · @Momibd (אינסטגרם) · @Momibd1 (טיקטוק)` (ראה סעיף הקרדיט). אווירת קיץ/חוף (רקע gradient תכלת-חול, לא תמונות כבדות).
- **Toolbar** דביק (`position: sticky; top:0`): חיפוש + צ׳יפים של אזור + צ׳יפים של עלות + מונה תוצאות ("מציג X מתוך 108") + כפתור נקה.
- **Grid כרטיסים**: כרטיס לכל פעילות. במובייל טור אחד, ב-desktop `auto-fill, minmax(300px,1fr)`.
- **כרטיס פעילות** מציג:
  - שם (מודגש, גדול)
  - badge אזור + badge עלות (עלות בצבע: חינם=ירוק, סמלי=כתום, בתשלום=כחול/אפור)
  - שורת מטא: `סוג · גילאים · משך` (אייקונים אופציונליים: 📍🧒⏱)
  - `למה כדאי` (פסקה)
  - `⭐ הטיפ של מורן` (בתיבה מודגשת בצד/רקע עדין)
- **Empty state**: כשאין תוצאות — "לא נמצאו פעילויות. נסו לנקות סינון." + כפתור נקה.
- **Footer**: קרדיט חוזר למורן בן דוד + לינקים קליקבילים לאינסטגרם (`@Momibd`) ולטיקטוק (`@Momibd1`).

## נגישות ו-RTL
- כל ה-flex/grid חייב לעבוד ב-RTL (השתמש ב-`margin-inline`/`padding-inline`, לא left/right קשיח).
- inputs עם `aria-label`. צ׳יפים כ-`<button aria-pressed>`.
- ניגודיות טובה על רקע הקיץ (טקסט כהה על בהיר).
- מספרים/ספירות בעברית תקינה.

## אקסטרה (רק אם נשאר זמन, לא חובה)
- טאב/סקשן נפרד ל**צ׳קליסט קיץ** (המערך `checklist` ב-JSON) — רשימת פריטים להורדה/הדפסה. תת-משני.
- כפתור "הדפסה" (`window.print()`) עם print stylesheet שמסתיר את ה-toolbar.

## מה לא לעשות
- לא להוסיף פעילויות/מחירים/כתובות שלא ב-JSON (אין ב-PDF כתובות מדויקות — אל תמציא).
- לא framework, לא CDN, לא build, לא storage APIs.
- לא לשבור RTL עם ערכי left/right קשיחים.

## פרסום ל-GitHub Pages (דומיין: bighofesh.natyharary.yachts)

לאחר שהאתר עובד מקומית, פרסם אותו כ-GitHub Pages תחת תת-הדומיין `bighofesh.natyharary.yachts`.

1. **CNAME** — צור קובץ בשם `CNAME` (בלי סיומת) בשורש ה-repo שהתוכן שלו שורה אחת בדיוק:
   ```
   bighofesh.natyharary.yachts
   ```
   GitHub Pages קורא את הקובץ הזה ומגדיר אוטומטית את ה-custom domain. אל תשכח לכלול אותו ב-commit.

2. **מבנה ה-repo** — `index.html` + `CNAME` בשורש. אם משתמשים ב-`activities.json` נפרד — הטמעת הנתונים ב-HTML (`const DATA`) עדיפה כדי שאין תלות ב-fetch; אבל אם הקובץ קיים בשורש הוא ייכלל אוטומטית.

3. **הפעלת Pages** — Settings → Pages → Source: "Deploy from a branch" → branch `main`, folder `/ (root)`. (אם ה-repo כבר מוגדר ל-Pages, רק ודא שה-branch נכון.)

4. **DNS (חד-פעמי, בצד הרושם/ספק של natyharary.yachts)** — אם עדיין לא מוגדר, צריך רשומת CNAME:
   ```
   bighofesh  CNAME  <username>.github.io.
   ```
   (החלף `<username>` בשם המשתמש/ארגון של ה-GitHub Pages.) המשתמש ציין שהדומיין כבר מארח GitHub Pages, כך שסביר שה-DNS כבר קיים — אם ה-subdomain החדש לא עולה, זו הרשומה שחסרה.

5. **HTTPS** — לאחר שה-domain מאומת ב-Settings → Pages, סמן "Enforce HTTPS". ההנפקה של תעודת ה-TLS יכולה לקחת כמה דקות.

6. **פקודות git** (אם זה repo חדש):
   ```bash
   git init && git add index.html CNAME
   git commit -m "מדריך חופש הגדול 2026"
   git branch -M main
   git remote add origin git@github.com:<username>/<repo>.git
   git push -u origin main
   ```

**אימות פרסום:** אחרי ~1–2 דקות, `https://bighofesh.natyharary.yachts` טוען את האתר עם HTTPS תקין ו-RTL עברי מלא.

> הערה: `.yachts` הוא TLD חוקי, אז הדומיין תקין. אם `dir=rtl` או הגופן העברי נראים שבורים ב-Pages אבל תקינים מקומית — כמעט תמיד זה בעיית encoding: ודא `<meta charset="utf-8">` ושהקובץ נשמר כ-UTF-8.

## אימות סיום
- נפתח מ-`file://` ועובד.
- 108 כרטיסים כשאין סינון; המונה תואם.
- בחירת "מרכז" → 16 כרטיסים; הוספת "חינם" מצמצמת בהתאם.
- חיפוש "מים" מחזיר פארקי מים / נחלים וכו׳.
- נראה תקין במובייל (טור אחד) ובדסקטופ (grid).
- הקרדיט למורן בן דוד + הלינקים לאינסטגרם וטיקטוק נראים וקליקבילים גם בראש וגם בתחתית (וגם באמצע גלילה).
- מופיע טקסט גלוי שמבהיר שהתוכן הוא של מורן בן דוד ושנתי הררי בנה/ה את האתר להנגשה וחיפוש בלבד.
